# PhoneHub Deployment Guide - Supabase + Vercel

## Overview
This guide walks you through hosting PhoneHub on Supabase (database) and Vercel (frontend).

---

## Step 1: Create the Database Table in Supabase

### 1.1 Go to Supabase Dashboard
- Visit [supabase.com](https://supabase.com)
- Sign in to your project
- Go to **SQL Editor**

### 1.2 Create Service Requests Table
Click **New Query** and paste this SQL:

```sql
-- Create service_requests table
CREATE TABLE service_requests (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  firstname VARCHAR(100) NOT NULL,
  email VARCHAR(100) NOT NULL,
  tel VARCHAR(20) NOT NULL,
  devicetype VARCHAR(50) NOT NULL,
  devicemodel VARCHAR(100) NOT NULL,
  repairtype VARCHAR(50) NOT NULL,
  description TEXT NOT NULL,
  servicedate DATE NOT NULL,
  submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create indexes for performance
CREATE INDEX idx_service_requests_email ON service_requests(email);
CREATE INDEX idx_service_requests_submitted_at ON service_requests(submitted_at DESC);

-- Enable Row Level Security
ALTER TABLE service_requests ENABLE ROW LEVEL SECURITY;

-- Allow anonymous public insert
CREATE POLICY "Allow public insert" ON service_requests
  FOR INSERT WITH CHECK (true);

-- Allow public read
CREATE POLICY "Allow public read" ON service_requests
  FOR SELECT USING (true);
```

Click **Run** ✓

## Step 4: Deploy to Vercel (Frontend)

### 4.1 Prepare GitHub Repository

First, push your files to GitHub:

```bash
# Initialize git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Add PhoneHub to Supabase integration"

# Add GitHub remote
git remote add origin https://github.com/nujoka1/phone-repair.git

# Push to GitHub
git push -u origin master
```

### 4.2 Deploy to Vercel

**Option A: Using Vercel CLI**

```bash
# Install Vercel CLI globally
npm install -g vercel

# Deploy from your project directory
vercel

# Follow the prompts:
# - Link to existing project? → No
# - Project name? → phonehub (or your choice)
# - Framework? → Other (since it's static HTML/CSS/JS)
# - Deploy to production? → Yes
```

**Option B: Using Vercel Dashboard**

1. Go to [vercel.com](https://vercel.com)
2. Click **Create Project** → **Import Git Repository**
3. Select your GitHub repository
4. Choose **Other** for framework
5. Click **Deploy**

### 4.3 Get Your Vercel URL

After deployment, Vercel will show your live URL:
```
https://phonehub-xxxxx.vercel.app
```

## Troubleshooting

### Problem: "Supabase not configured"
- Check that `SUPABASE_URL` and `SUPABASE_ANON_KEY` are correct (no extra spaces)
- Ensure they're pasted exactly as shown

### Problem: Form doesn't submit
- Open browser console (`F12` → Console tab)
- Look for error messages
- Verify credentials are filled in

### Problem: Can't see data in Vercel
- Check browser console for errors
- Verify Supabase RLS policies are correct (should allow public insert/read)
- Try submitting again

### Problem: Delete/View buttons don't work
- Ensure you're logged in to Vercel (data is live)
- Check that Supabase connection is active

## Security Notes ⚠️

1. **Anon Key is Public**: It's safe to expose (restricted to specific operations)
2. **RLS Policies**: Currently allow public access. For production, consider adding authentication
3. **Environment Variables** (Advanced): Use Vercel Environment Variables instead of hardcoding:
   - Go to Vercel → Settings → Environment Variables
   - Add `VITE_SUPABASE_URL` and `VITE_SUPABASE_KEY`
   - Update JavaScript to use `process.env.VITE_SUPABASE_URL`
